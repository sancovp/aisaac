# The Claude Code Course Pattern — the Universal Template, bound to Claude Code

The Universal Course Template with its variables **bound to a Claude-Code-primitives course**: you teach the
generic Claude Code primitive, then show YOUR harness (`{system}`) doing it right-and-forever, climbing the
complexity ladder, peaking at the harness/knowledge-graph. Worked instance: `{system}` = **doc-mirror**
(the GTM course's instance was `{system}` = a GTM-OS — same pattern, different harness).

## The bound variables

- **`{core_thesis}`** = *"Context is the moat / the harness beats the model / every primitive exists to SAVE
  context."*  Supporting theses: progressive disclosure; **a feedback loop is what turns a tool into a
  system**; CLIs-over-MCPs (load context on demand, don't flood it); a-place-for-everything / never-delete-only-archive.

- **`{primitives[]}`** = the Claude Code primitives in **complexity-ladder order** (the spine):
  1. **Setup** — the interface (Cursor/terminal) + the agent + model choice (incl. a cheap escape-hatch model)
  2. **Git / source-of-truth** — version-control-as-safety ("time machine / save-scum"); issues = handoffs; projects = Kanban
  3. **Monorepo + env/keys** — a place for everything; per-area sub-repo + its own CLAUDE.md; no context pollution
  4. **The Layers + `{core_thesis}`** — terminal→files→agents→output; "context is the moat = your taste, programmed in"
  5. **CLAUDE.md + rules** — the brain; keep it LEAN; standing rules; only grow it from session-log review
  6. **Skills** — *a skill = a prompt-with-frontmatter = one SOP*; trigger words; workflow + quality gates + anti-patterns; chaining
  7. **CLIs vs MCPs** — the load-bearing choice: CLIs loaded on demand vs MCP context-rot ("almost always want a CLI")
  8. **Arguments + Hooks** — args unite separated systems + hide keys; **hooks = deterministic code that ENFORCES** (rules are just words)
  9. **Agents / sub-agents** — the org chart; orchestrator spawns parallel specialists; independent context windows; spend control
  10. **The feedback / auto-research loop** — the HIGHEST-LEVERAGE system: one data point → ground truth → anneal N passes ("tool→system")
  11. **Automate vs agentic vs run-yourself** — the deployment ladder (interactive → cron/queue → managed → full harness)
  12. **Plugins / marketplaces** — multiplayer; publish your way of working; permissions/security
  13. **THE CLIMAX — the knowledge-graph / context-OS / the harness** — *"the most important thing"*; traceable outputs; "the harness beats the model"

- **`{system}`** = YOUR advanced implementation, shown at EVERY concept beat. For us = **doc-mirror** (+
  CartON the knowledge graph, the cursor, the journal, the prompt-skill store, `ship-a-plugin`, the nomicon).
  Per-concept beat: *"this is `<CC primitive>`, here's how it works + tips — but in **doc-mirror** we do it
  like THIS, because it solves `<that problem>` **forever**."*

- **`{builds[]}`** = interleaved live builds (build-beat) demonstrating the harness on a real task.
- **`{capstone_build}`** = *"boot/run your own `{system}` specialization end-to-end"* — for us: **boot your
  own doc-mirror specialization from prompts up the complexity ladder, the thing that then uses itself with
  you.**
- **`{climax_differentiator}`** = the harness / knowledge-graph (doc-mirror + CartON) — the moat.
- **`{lead_magnets[]}`** = the free companion repo + the plugin + free chapters · **`{hard_CTA}`** = the one
  buy/book at the capstone · **`{upsells[]}`** = higher tiers (done-with-you, the full system, the Me2B CAVE).

## The specialization rule
A **Claude Code Course** = the Universal Course Template where `{primitives[]}` = the CC-primitive ladder,
`{system}` = your harness shown at every concept beat, and the **climax = the harness itself**. The course
TEACHES Claude Code (the value freely given) and SELLS the harness (the product) through the gap. To
generate it: bind these variables, then run the **Course-Construction Loop** (`course-construction-loop.md`).

## Our instance (the doc-mirror course)
Bind `{system}` = doc-mirror and the spine above produces the doc-mirror course: each CC primitive taught,
then "here's how doc-mirror does it forever" (rule→skill→harness→plugin→OS = the same complexity ladder),
capstone = boot-your-own-doc-mirror, climax = doc-mirror + CartON as the knowledge-graph harness. This is the
`Course_And_Funnel` product vision + the `Course_Format` beat, made generative. Free chapters teach Claude
Code; the paid product is the doc-mirror that does all of it and uses itself with you.
