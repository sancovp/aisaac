# Course Construction Loop — Build a Module, Roll Up the Whole, Repeat

How to CONSTRUCT a course from the Universal Course Template: derive the spine, then build **one module at
a time**, and **roll up the entire course after each add** — so module N+1 is built on a *coherent whole*,
not in isolation. This is a progressive-disclosure climb: you cannot generate a coherent multi-module
course in one shot; you climb it module-by-module, re-grounding the whole at each step. (Each `module add ⟹
roll up again`.)

## Inputs
The filled Universal-Course-Template variables: `{transformation}`, `{core_thesis}` + `{supporting_theses[]}`,
`{primitives[]}`, `{system}`, `{builds[]}`, `{highest_leverage_system}`, `{capstone_build}`,
`{climax_differentiator}`, `{lead_magnets[]}`, `{hard_CTA}`, `{upsells[]}`.

## STEP 0 — Derive the SPINE
From the variables, produce the ordered module list:
- one CONCEPT module per `{primitive}` in beginner→advanced (ladder) order;
- interleave the BUILD modules (`{builds[]}`) at the points where a just-taught concept is ripe to demonstrate;
- place the THESIS module early, the HIGHEST-LEVERAGE-SYSTEM module mid-late, the CAPSTONE + CLIMAX late.
**Output:** the spine = titled modules, one line each, with: which KIND (concept/build beat), which
`{primitive}`/`{build}` it teaches, and which `{system}` artifact it shows.

## STEP 1 — Initialize the ROLLUP
Create the running **COURSE DOC** (the living whole): the hook/promise, `{core_thesis}`, the spine, the
**funnel-beat map** (lead magnets placed, the ONE hard CTA at the capstone, upsells after), and a
**"course-shape-in-one-paragraph."** Every module integrates into this doc.

## STEP 2 — THE LOOP (per module M, in spine order)
For each module M:
- **a. CONSTRUCT M fully** — apply its beat pattern (concept or build). Ground it in the real `{system}`
  artifact / `{primitive}` / `{build}`. Author the actual module content, not a stub.
- **b. ROLL UP** — integrate M into the COURSE DOC and re-ground the whole:
  - append M;
  - **re-check COHERENCE**: does M build on every prior module? does beginner→advanced still hold? does
    `{core_thesis}` thread through M's benefit beat? any forward-reference to fix or prerequisite to add?
  - **update the funnel-beat map** (did M add a lead magnet? is the hard CTA still ONLY at the capstone?);
  - **re-derive the "course-shape-in-one-paragraph"** so the whole still reads as ONE arc;
  - **if the rollup reveals an incoherence** (M doesn't build, a prereq is missing, the thesis broke):
    FIX the spine / prior modules BEFORE continuing. *The rollup is the gate — do not advance on an
    incoherent rollup* (this is the OK-stable-signal per module).
- **c. Continue to M+1** — now built on the coherent whole.

## STEP 3 — CLOSURE
Done when: every spine module is constructed AND integrated; the rollup reads as one coherent
beginner→advanced arc; `{core_thesis}` threads every module's benefit; the funnel beats are placed
(lead magnets throughout, ONE hard CTA at the capstone, upsells after); the CLIMAX lands as "the most
important thing." The COURSE DOC is now the finished course blueprint.

## Why roll up each time (not author-then-assemble)
A coherent course is a **climbed gradient, not a one-shot generation.** Each module's coherence is the
ground the next is built on; authoring all modules independently then stapling them yields drift
(forward-references, broken progression, a thesis that appears then vanishes). Rolling up after each add
forces the whole to stay admissible at every step — the same discipline as the framework CHAPTER pipeline
(each module ≈ a chapter) climbing the complexity ladder. Composes with:
`autorealization-and-progressive-disclosure` (climb, don't jump), the rollup pattern,
`braid-nonlinear-problem-solving` (the spine is the stage-DAG produced before walking it), and
`universal-course-template.md` (the shape each module fills).

## How to run it
This is runnable as a prompt-skill: feed the filled variables, run STEP 0–3, looping STEP 2 once per module.
It can be a single agent climbing the loop, or one agent per module that hands the rolled-up COURSE DOC to
the next (episodic composition through the durable COURSE DOC).
