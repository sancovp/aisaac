## Summary

Tommy Chryst presents a comprehensive course on building and selling AI Operating Systems (AIOS) - custom intelligence layers built around businesses that serve as the foundational infrastructure for all future AI implementations. The course teaches a framework called CRAFT (Context, Reach, Actions, Flow, Tuning) and demonstrates building an AIOS using Claude Code, with nodes as installable capability packages and skills as slash commands. Chryst positions AIOS as the single most powerful AI application right now, citing Coinbase and Block's public restructuring around this concept, and teaches both how to build one for personal productivity and how to sell these systems to small-to-medium businesses for $5,000-$10,000+ per engagement.

## Key Sections

### What is an AIOS?
- **Definition:** A custom intelligence layer built around your business - not an agency model, but something you build and deliver
- **Core premise:** Put your business in the middle and build AI infrastructure around it
- **Why now:** Three things converged - models are smart enough, agency/tool use is first-class and usable, and coding is democratized (AI does the technical work)
- **Business thesis:** Companies like Coinbase (14% layoffs) and Block (Jack Dorsey) are restructuring around this concept, treating AIOS as the core of future business operations

### The CRAFT Framework
- **Context:** What the AI knows (static data) - business info, voice, goals, strategy stored in files like `context.md`
- **Reach:** What the AI can see (live/malleable data) - APIs, emails, YouTube metrics, call transcripts that change daily
- **Actions:** What the AI can actually do - skills like generating thumbnails, scripts, morning briefings
- **Flow:** How things run - scheduling, manual triggering, mobile access via Telegram
- **Tuning:** How the system grows - auditing, removing unused features, adding new automations

### Technical Infrastructure
- **Claude Code (paid):** Core AI agent that handles all technical work via plain English
- **IDE (Google Anti-Gravity/VS Code/Cursor):** Required for the full Claude Code experience
- **GitHub:** Backs up the entire AIOS to the cloud, enables version history and cross-device access
- **Telegram:** Enables mobile access to the AIOS for on-the-go business management
- **API Keys vs MCP:** Prefers API keys as they use less context than MCP connections

### The Six Core Nodes

1. **Vault Node:** Creates workspace structure, folder organization, and foundation files that other nodes attach to
2. **Context Node:** Fills in deeper business knowledge - who customers are, what you sell, communication style, current goals. Allows importing existing Claude/ChatGPT chat history
3. **Intelligence Node:** Connects to live business data - YouTube (via Vid IQ), Stripe, Gmail, Outlook, Fireflies (meeting transcripts). Pulls data on schedules, updates snapshot files
4. **Mobile Node:** Creates Telegram bot connected to the workspace for mobile access (stateless - single questions only)
5. **Coffee Debrief:** Automated morning intelligence report sent to Telegram - compiles metrics, filters emails, summarizes meetings, delivers before the day starts
6. **Productivity Node:** GTD-style task management system with inbox, projects, next actions, waiting for, someday/maybe lists, and weekly review cadence

### Skills (Slash Commands)
- **/start:** Initializes session, reads all context files, gets agent ready
- **/pulse:** Returns live snapshot of business numbers vs. goals
- **/install:** Installs nodes by copying path from GitHub
- **/process:** Clears task inbox by routing items to projects/next actions
- **/audit:** Scores AIOS completeness across all CRAFT layers
- **/tune:** Provides recommendations for what to automate/improve next

### The Operator Trap
- **Problem:** Founders spend too much time "in the business" (1x activities like meetings, admin, reporting) instead of "on the business" (growth initiatives)
- **Historical solution:** Hire COO ($150k+/year, hard to find right person)
- **AIOS solution:** Automates the "in the business" work, inverts time allocation to 80% on business
- **Goal:** 60-70% of recurring personal tasks automated within 30 days

### Automation Spectrum
Every task lives somewhere on a spectrum from ignore to fully automated:
- **Ignore:** AI knows it exists but dismisses it (basic ChatGPT usage)
- **Draft/Augment:** AI does most of the task, human edits
- **Automated:** Task completes end-to-end without human input
- **Goal:** Keep moving tasks right on this spectrum

### Selling the AIOS Offer

**Two Model Approaches:**
1. **Consultative/Custom:** $5,000-$10,000 for CEO setup, $1,000-$2,000+ per additional employee
2. **Productized:** Industry-specific pre-built solutions (e.g., e-commerce on Shopify) at lower price but higher volume

**Key Positioning:**
- Not "adding AI to your business" - this fails
- Instead: "Foundation for how you run the company" where everything future builds on top
- AIOS becomes the circular center of a flattened org structure (not the traditional tree/wedding cake hierarchy)
- Upsells include voice agents, CRM builds, chatbots, outbound sales automation, custom workflows

**Audit First:**
- Run paid audits ($500-$3,000 depending on company size) before building
- Audits prevent unpaid consulting baked into fixed-price projects
- Audit output is the AIOS proposal itself
- Covers: current tools, data sources, team time allocation, workflow mapping, automation opportunities

**Ideal Client Profile:**
- **Sweet spot:** 10-50 employee businesses doing $5M+ annually
- **Characteristics:** Looking to scale, CEO still in the weeds (feels value immediately), local or regional businesses
- **Higher value:** Online-native businesses (more data artifacts - emails, Slack, CRM, analytics, call transcripts)
- **Avoid initially:** Solopreneurs (won't pay high prices), enterprise (too hard to close initially)

### Naval's Leverage Framework
- **Labor:** Expensive, requires finding/hiring the right people
- **Capital:** Requires existing capital to deploy
- **Media:** Podcast, YouTube - asymmetric but doesn't fit most business owners
- **Code:** Most powerful leverage - now democratized and accessible to non-technical people via AI coding agents. AIOS gives access to code leverage for ~$20/month

## Competitive Intelligence (for Isaac)

**What is Tommy selling?**
- A course teaching how to build and sell AI Operating Systems
- Free community + GitHub repository with node templates
- Consulting/services: Building AIOS for SMBs at $5k-$10k+
- Positioning himself as a thought leader in the AIOS space

**How does this compare to what we build?**
- **Similarities:** Both focus on AI infrastructure for businesses, both leverage Claude Code, both emphasize context/data as the core differentiator
- **Differences:** Tommy is teaching the methodology as a course + service business. His GitHub nodes are templates that require manual setup. The Coffee Debrief is hardcoded to specific use cases. His approach is heavily focused on "selling" as the primary monetization.
- **Isaac's focus:** We likely focus on actual productization, automated delivery, and deeper integrations. Tommy's course is more "teach a person to fish" for building their own or selling the service.

**What can we learn?**
- The CRAFT framework is a solid mental model for structuring AIOS layers
- The node/skill architecture is clever for modularity and reusability
- The "audit first, then build" sales approach is smart for high-ticket services
- The Coffee Debrief concept is highly valuable and replicable
- The Telegram integration for mobile access is a practical flow addition
- GitHub backing for version control is essential infrastructure

**What are they wrong about?**
- **API vs MCP stance:** His preference for API keys over MCP is situational - MCP is often the only option and is appropriate when you need deep tool integration. Also underestimates MCP's benefits for complex automation.
- **Windows scheduling:** Suggests leaving computer on overnight for task scheduling - this is fragile. Better approach: dedicated compute or proper cron/scheduler setup.
- **Mobile node limitations:** Admits the Telegram bot is "stateless" as a limitation rather than building session memory into it.
- **Pricing model:** $5-10k for CEO setup feels underpriced for the value delivered if truly implemented properly. The "fork environment for each employee" approach creates maintenance nightmares at scale.
- **No actual product:** Everything taught requires manual setup, configuration, and maintenance. No mention of automated provisioning, self-service options, or SaaS-ification.

## Actionable Takeaways

1. **Adopt the CRAFT framework** for structuring any AIOS we build - it cleanly separates the layers and provides a progression for implementation (Context → Reach → Actions → Flow → Tuning)

2. **Build a node-based architecture** similar to Tommy's GitHub structure - modular, installable capability packages that can be mixed and matched. This enables both custom builds and productized offerings.

3. **Implement the Coffee Debrief pattern** as a flagship automation feature - automated morning reports delivered to mobile summarizing metrics, emails, and priorities. This is immediately valuable and highly demonstrable.

4. **Add Telegram (or Discord) integration** for mobile access to AIOS - this addresses the "away from desk" metric and enables true mobile business management. Build in session memory rather than accepting stateless limitations.

5. **Create an audit-first sales process** for any service engagements - run paid discovery audits before committing to fixed-price builds. This prevents scope creep and frames the build as the deliverable from the audit.

6. **Target 10-50 employee SMBs** as the ideal client profile - they're large enough to have real revenue/ workflows, small enough for the CEO to feel immediate value and make quick decisions.

7. **Build self-service/template options** - Tommy's course fills a gap between free YouTube content and expensive custom builds. We could productize the node templates into a self-serve platform or lower-tier offering.

8. **Use GitHub for version control** on all AIOS configurations - this enables rollback, collaboration, and client handoff. Essential infrastructure that Tommy correctly emphasizes.

9. **Measure "away from desk" autonomy** as a success metric - can the client run their business from their phone? This is more valuable to business owners than traditional productivity metrics.

10. **Position AIOS as organizational infrastructure**, not as "adding AI" - the framing matters. We're building the foundation for how the company operates, not bolting on another tool.
