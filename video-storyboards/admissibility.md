## Scene 2: The Category Shift (0:15 - 1:30)
**Visual:** [AVATAR: GNOSYS, direct gaze, pointing for emphasis] → [CUT TO: Excalidraw diagram animating in] → [AI-IMAGE: A glowing boundary membrane surrounding a code repository, with red warning lines marking hidden callers and side effects, dark background with cyan wireframe grid]
**Voiceover:** "The hype world says give the agent better context. But better context is not more context. Better context is context whose boundary, authority, and hidden causal edges are known. A giant context window full of stale docs, partial search results, and false comments makes the model WORSE. The invariant is simple: no transformation without a bounded witness path. For code — entrypoint known, execution path traced, hidden calls exposed. For business automation — SOP boundary known, owner known, exception paths known. Different domains, same invariant."

## Scene 3: Bridging the Hype (1:30 - 3:00)
**Visual:** [AVATAR: GNOSYS, debunking mode] → [AI-IMAGE: A shattered glass floor revealing layers of invisible dependencies beneath, with golden light illuminating the hidden connections, noir style]
**Voiceover:** "Let's bridge some hype. 'Prompt engineering is dead' — no, it became one interface inside context control. 'MCP and tools make agents powerful' — tools increase action capacity but not action UNDERSTANDING. A tool call can trigger hidden effects — file edit, hook fires, state changes, index updates. That's why we need action causality maps. 'Agents need harnesses' — yes, but a harness without admissibility gates can reliably execute inadmissible actions. That's very dangerous and very common. 'Agents fail because they lose context' — sometimes. But often they have the information, they just fail to keep it operationally active. That's a concentration problem, not a context problem."
**Remotion animation:** Each hype claim appears as a speech bubble that gets partially struck through, with the "deeper correction" appearing below in amber text

## Scene 4: Catastrophe Engineering (3:00 - 4:15)
**Visual:** [AI-IMAGE: An agent standing at a crossroads where each path is marked with typed semantic markers glowing in amber and violet, futuristic control room aesthetic] → [REMOTION-ANIMATION]
**Voiceover:** "Catastrophe Engineering designs failures so they improve the system. Step 1 — name the invalid frame. Step 2 — expose the missing boundary object. Step 3 — supply the higher-order frame. Step 4 — add a persistent witness or gate. Step 5 — future agents encounter the improved boundary BEFORE repeating the failure. Every recurring mistake should be engineered into a surface that forces the agent to fold into the space where the mistake becomes obvious."
**Remotion animation:** A failure event (red flash) → transforms into a named error → spawns a rule/gate → future path now has a glowing checkpoint that catches the same class of error
