## Scene 2: The Assembly Lie (0:15 - 1:15)
**Visual:** [AVATAR: GNOSYS, wry smile] → [AI-IMAGE: Five perfect geometric shapes that individually glow with inner light but when pushed together create jagged misaligned seams with sparks at every boundary, dark technical illustration] → [CUT TO: Excalidraw diagram]
**Voiceover:** "Component A works perfectly. Component B — flawless. C, D, E — all tested, all green. Then you plug them together. Disaster. Interfaces mismatch. Assumptions collide. Errors in A look like success to B. The same field name means different things on different sides. Composition is NOT assembly. Individual component quality doesn't guarantee composition quality."
**Remotion animation:** Five glowing shapes sliding together — sparks and error flashes at every boundary, the combined shape clearly broken despite each piece being individually perfect

## Scene 3: Five Requirements (1:15 - 2:45)
**Visual:** [AVATAR: GNOSYS counting] → [REMOTION-ANIMATION: checklist]
**Voiceover:** "Successful composition requires five things. Interface clarity — not just types, semantics. What does this input MEAN? Assumption alignment — conflicting assumptions are time bombs. Boundary protocols — where A ends and B begins must be explicit. Error propagation — when A fails, what does B see? If A's failure looks like success to B, you're broken invisibly. Semantic compatibility — not just types match but meanings match. user_id could mean database row ID to one side and auth provider external ID to the other."
**Remotion animation:** Five checkboxes appearing with labels, each one getting a green check as explained — then showing what happens when any one is unchecked (red flash, system breaks)
