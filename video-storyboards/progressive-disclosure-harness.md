## Scene 2: The Drowning Problem (0:15 - 1:15)
**Visual:** [AVATAR: GNOSYS, incredulous] → [REMOTION-ANIMATION: agent overwhelmed]
**Voiceover:** "Here's the thing about AI agents in production: they get WORSE the more capable you make them. You give the agent 20 integrations and 300 tools. It starts hallucinating tools that don't exist. Forgets what it called 30 seconds ago. Three steps into a workflow it stops and asks 'what should I do next?' — like a surgeon putting down the scalpel mid-operation to ask what organ they're on. Every additional capability makes ALL capabilities work worse. Giving the agent everything it might need does not scale."
**Remotion animation:** An agent icon with tools piling around it — 5 tools (green, working), 50 tools (yellow, struggling), 300 tools (red, confused, hallucinating ghost tools)

## Scene 3: The Solution (1:15 - 2:45)
**Visual:** [CUT TO: Excalidraw diagram animating] → [AVATAR: GNOSYS, solution mode]
**Voiceover:** "The fix is progressive disclosure. Show the agent what it needs WHEN it needs it. Not everything at once — the relevant subset for the current step. The harness encodes your SOPs as enforceable constraints, not suggestions. Step 1 of the workflow? The agent sees only the tools for step 1. Step 2? Tools swap. The agent never sees 300 options. It sees 5 relevant ones. This is the same principle as a well-designed UI. You don't show every button on the homepage. You reveal complexity as the user goes deeper."
**Remotion animation:** A workflow with steps 1-5, at each step a different small set of 3-5 tools appears, previous ones fading away — showing the progressive reveal

## Scene 4: Outro (2:45 - 4:00)
**Visual:** [AVATAR: GNOSYS, wrapping up]
**Voiceover:** "The harness does three things. Prevents capability overwhelm — agent only sees what's relevant. Enforces ordering — can't skip steps or call tools in wrong sequence. Provides concentration — agent's attention stays on the current task, not the full toolkit. This is the difference between an agent that completes 10-step workflows reliably and one that falls apart at step 3. Not model quality. Architectural discipline. Full post below."
