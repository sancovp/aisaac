## Scene 2: The Problem (0:15 - 1:00)
**Visual:** [AVATAR: GNOSYS, incredulous] → [CUT TO: Excalidraw diagram]
**Voiceover:** "Your team gave the agent 30 tools. A knowledge graph. A deployment pipeline. Everything it needs. And it still drifts. Makes invalid moves. Calls tools in wrong order. But that's exactly what most agent architectures do with tools."
**Remotion animation:** Chess pieces scattered randomly on a board with no grid — then a grid appears and suddenly the pieces know where to go

## Scene 3: SHELL Framework (1:00 - 2:30)
**Visual:** [AVATAR: GNOSYS, teaching mode] → [REMOTION-ANIMATION: game compilation]
**Voiceover:** "SHELL compiles a toolkit into a playable game. The agent doesn't just have tools — it has rules, valid moves, win conditions, and loss conditions. What are the boundaries? What's a valid move vs invalid? What does winning look like? What does losing look like? When is the game over? This is the compilation step everyone skips. Without it, the agent is playing Calvinball — making up rules as it goes. With it, the agent has a game it knows how to play. And games are what agents are actually good at."
**Remotion animation:** A pile of tools being fed into a "compiler" that outputs a clean game board with rules, valid moves highlighted, and clear win/loss conditions
